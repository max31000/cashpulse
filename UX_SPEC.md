## 2. Общий Layout (AppShell)

Используется `AppShell` из Mantine v7 с фиксированным `Navbar` на десктопе и `Drawer`-меню на мобильных.

### 2.1 Структура AppShell

```
┌──────────────────────────────────────────────────────┐
│  [Sidebar 250px]  │  [Main Content]                  │
│                   │                                  │
│  (десктоп ≥768px) │  padding: 16px                   │
└──────────────────────────────────────────────────────┘
```

**Десктоп (≥ 768px):**
- `AppShell.Navbar` — ширина `250px`, фиксированная (не скроллится), высота 100vh
- `AppShell.Header` — **скрыт** на десктопе (`hidden` на `≥ sm`)
- `AppShell.Main` — занимает оставшееся пространство, `padding="md"` (16px)

**Мобильные (< 768px):**
- `AppShell.Header` — высота `60px`, отображается
  - Слева: `Burger`-иконка (открывает Drawer)
  - Центр: логотип "CashPulse" (текст, уменьшенный)
  - Справа: пусто (зарезервировано)
- `AppShell.Navbar` — скрыт, вместо него `Drawer` из Mantine
  - Drawer открывается слева, ширина `250px`
  - Закрывается по клику на overlay или по нажатию крестика
- `AppShell.Main` — полная ширина, `padding="sm"` (12px)

### 2.2 Sidebar (Navbar)

Sidebar — вертикальный flex-контейнер с `justify-content: space-between`.

**Верхняя секция:**

```
┌─────────────────────────────┐
│  ≋ CashPulse                │  ← Логотип
├─────────────────────────────┤
│  📊  Дашборд                │  ← NavLink, route "/"
│  💰  Операции               │  ← NavLink, route "/operations"
│  🏦  Счета                  │  ← NavLink, route "/accounts"
│  🔮  Сценарии               │  ← NavLink, route "/scenarios"
│  🏷  Теги                   │  ← NavLink, route "/tags"
│  ⚙   Настройки              │  ← NavLink, route "/settings"
└─────────────────────────────┘
```

**Логотип:**
- Компонент: `Group` с `gap="xs"`
- Иконка: SVG-волна или иконка пульса из Tabler Icons (`IconActivity` или `IconHeartbeat`), размер `28px`, цвет `theme.primaryColor`
- Текст: "CashPulse", `fw={700}`, `fz="xl"`, цвет — primary цвет темы
- Клик по логотипу → навигация на `/`
- `padding="md"` вокруг логотипа, `mb="md"` снизу, `Divider` после логотипа

**Навигационные ссылки:**
- Компонент: `NavLink` из Mantine v7
- Props: `component={Link}` (React Router), `to={route}`
- `active` — определяется через `useMatch` (активен если текущий route совпадает)
- Активный элемент: `variant="filled"` (фон primary цвет, текст белый)
- Неактивный: `variant="subtle"`, hover — лёгкая подсветка фона
- `leftSection` — эмодзи или Tabler-иконка, размер `20px`
- Padding: `px="md"`, `py="xs"` для каждой ссылки
- `fz="sm"`, `fw={500}`

**Нижняя секция (Footer Sidebar):**
- `marginTop="auto"` — прижато к низу
- `Divider` перед footer
- Текст: "v0.1.0", `fz="xs"`, `c="dimmed"`, `ta="center"`, `py="sm"`
- Кнопки авторизации **отсутствуют** (MVP без авторизации)

### 2.3 Активный элемент навигации

Используется `useMatch` из React Router для точного совпадения:
- `/` — exact match (`end: true`)
- Все остальные — prefix match

## 3. Глобальные компоненты и паттерны

### 3.1 Notifications (Toast)

- Компонент: `Notifications` из `@mantine/notifications`
- Позиция: `top-right`
- Авто-закрытие: 5000ms (5 секунд)
- Ошибки API: `color="red"`, иконка ❌
- Успех: `color="green"`, иконка ✓
- Инфо: `color="blue"`, иконка ℹ️

### 3.2 Модальные окна

- Компонент: `Modal` из Mantine
- По умолчанию: `centered={true}`, `closeOnClickOutside={false}` (во избежание случайного закрытия)
- Большие формы: `size="lg"` (600px)
- Подтверждения: `size="sm"` (400px)
- `overlayProps={{ blur: 3 }}` — размытие фона

### 3.3 Диалог подтверждения удаления

Используется `modals.openConfirmModal` из `@mantine/modals`:
- Заголовок: "Подтвердите удаление"
- Текст: "Это действие необратимо. Продолжить?"
- Кнопка подтверждения: `color="red"`, текст "Удалить"
- Кнопка отмены: `variant="subtle"`, текст "Отмена"

### 3.4 Цветовая схема статусов

| Статус | Цвет Mantine | Применение |
|---|---|---|
| Успех / Доход | `green` | Положительные суммы, подтверждённые операции |

| Ошибка / Расход | `red` | Отрицательные суммы, критические алерты |
| Информация | `blue` | Плановые операции, инфо-алерты |
| Нейтральный | `gray` / `dimmed` | Вспомогательный текст, архивные записи |

### 3.5 Типы счетов — цвета Badge

| Тип | Цвет Badge | Текст |
|---|---|---|
| debit | `blue` | Дебетовый |
| credit | `orange` | Кредитный |
| investment | `green` | Инвестиционный |
| cash | `gray` | Наличные |

## 4. Форматирование данных

Все форматирования выполняются через утилитарные функции, вызываемые единообразно во всём приложении.

### 4.1 Валюты

```
RUB → Intl.NumberFormat('ru-RU', { style: 'currency', currency: 'RUB', maximumFractionDigits: 0 })
      Пример: "150 000 ₽"
      Отрицательная: "−95 000 ₽" (знак минус, не дефис)

USD → "$1,200" (Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' }))

EUR → "€800" (Intl.NumberFormat('de-DE', { style: 'currency', currency: 'EUR' }))

Другие → "{amount} {CODE}"
          Пример: "50 000 CNY"
```

Знак суммы в UI:
- Доход (Amount > 0) → "+150 000 ₽", цвет `green`
- Расход (Amount < 0) → "−95 000 ₽", цвет `red`
- Отображается без знака там, где контекст ясен (например, в колонке "Доходы" таблицы)

### 4.2 Большие числа (сокращённые)

```
≥ 1 000 000 → "1.5М ₽"
≥ 1 000     → "150к ₽"
< 1 000     → "850 ₽"
```

Используется для осей графиков и компактных индикаторов.

### 4.3 Даты

| Формат | Пример | Применение |
|---|---|---|
| Полная | "15 января 2026" | Заголовки секций, детальные карточки |
| Compact | "15 янв" | Операции в ленте (compact view) |
| Месяц-год | "Январь 2026" | Таблица по месяцам |
| Месяц краткий | "янв '26" | Ось X на графиках |
| Относительная | "через 3 дня", "вчера", "сегодня" | Ближайшие операции (если ≤ 7 дней) |
| День недели + дата | "15 января 2026 · Вторник" | DateHeader в ленте операций |

Все даты хранятся в UTC, отображаются в локальном времени браузера.  
Локаль для дат: `ru-RU`, используется `Intl.DateTimeFormat`.

### 4.4 Числа

```
Проценты: "12.5%" (toFixed(1))
Количество дней: "7 дней" / "1 день" (с правильным склонением: день/дня/дней)
```

### 5.3 Forecast Chart

**Компонент:** `Paper` (`p="lg"`, `radius="md"`, `shadow="sm"`, `withBorder`) содержащий:
1. Заголовок с контролами
2. Recharts AreaChart

**Заголовок секции:**
```
┌─────────────────────────────────────────────────────┐
│  Прогноз баланса          [3 мес][6 мес][12 мес]   │
└─────────────────────────────────────────────────────┘
```
- Заголовок: "Прогноз баланса", `fz="sm"`, `fw={600}`, `c="dimmed"`, `tt="uppercase"`
- `SegmentedControl` выровнен по правому краю (`Group justify="space-between"`)
- Значения: `"3 мес"`, `"6 мес"`, `"12 мес"`; default: `"6 мес"`
- При переключении — перезапрос Forecast API с новым горизонтом

**Recharts AreaChart — конфигурация:**

```jsx
<ResponsiveContainer width="100%" height={300}>
  <AreaChart data={timelineData} margin={{ top: 10, right: 10, bottom: 0, left: 10 }}>
    <defs>
      {/* Градиент для позитивной зоны (> 50000) */}
      <linearGradient id="gradGreen" x1="0" y1="0" x2="0" y2="1">
        <stop offset="5%" stopColor="rgba(34, 197, 94, 0.6)" />
        <stop offset="95%" stopColor="rgba(34, 197, 94, 0.05)" />
      </linearGradient>
      {/* Градиент для предупредительной зоны (0–50000) */}
      <linearGradient id="gradYellow" x1="0" y1="0" x2="0" y2="1">
        <stop offset="5%" stopColor="rgba(234, 179, 8, 0.6)" />
        <stop offset="95%" stopColor="rgba(234, 179, 8, 0.05)" />
      </linearGradient>
      {/* Градиент для отрицательной зоны (< 0) */}
      <linearGradient id="gradRed" x1="0" y1="0" x2="0" y2="1">
        <stop offset="5%" stopColor="rgba(239, 68, 68, 0.6)" />
        <stop offset="95%" stopColor="rgba(239, 68, 68, 0.05)" />
      </linearGradient>
    </defs>
    <XAxis dataKey="date" tickFormatter={formatXAxisDate} />
    <YAxis tickFormatter={formatYAxisAmount} />
    <CartesianGrid strokeDasharray="3 3" opacity={0.3} />
    <Tooltip content={<CustomForecastTooltip />} />
    
    {/* Опорные линии */}
    <ReferenceLine y={0} stroke="rgba(239, 68, 68, 0.8)" strokeDasharray="4 4" />
    <ReferenceLine y={50000} stroke="rgba(234, 179, 8, 0.8)" strokeDasharray="4 4" />

    {/* Основная линия прогноза */}
    <Area
      type="monotone"
      dataKey="balance"
      stroke={dynamicStrokeColor} // green/yellow/red в зависимости от текущего значения
      fill="url(#gradGreen)"      // градиент заливки — сменяется динамически
      strokeWidth={2}
      dot={false}
      activeDot={{ r: 5 }}
    />
    
    {/* Линия активного сценария (если есть) */}
    {hasActiveScenario && (
      <Line
        type="monotone"
        dataKey="scenarioBalance"
        stroke="#3B82F6"
        strokeDasharray="8 4"
        strokeWidth={2}
        dot={false}
      />
    )}
  </AreaChart>
</ResponsiveContainer>
```

**X-ось:**
- `tickFormatter`: функция `formatXAxisDate` — `Intl.DateTimeFormat('ru-RU', { month: 'short', year: '2-digit' })`, пример: "янв '26"
- Тики: каждый месяц (или каждые 2 месяца при горизонте 12 мес для читаемости)
- `tick={{ fontSize: 11 }}`, `c="dimmed"`

**Y-ось:**
- `tickFormatter`: функция `formatYAxisAmount` — сокращённый формат: "150к ₽", "1.5М ₽"
- `width={65}` (для читаемости меток)

**Динамический цвет линии и заливки:**

Поскольку Recharts не поддерживает нативно смену цвета линии по значению, используется следующий подход:
- Вычислить минимальное значение в данных: если `min < 0` → использовать красный `fill="url(#gradRed)"` и `stroke="#EF4444"`
- Если `min >= 0` и `min < 50000` → жёлтый `fill="url(#gradYellow)"` и `stroke="#EAB308"`
- Если `min >= 50000` → зелёный `fill="url(#gradGreen)"` и `stroke="#22C55E"`
- **Примечание:** цвет определяется по минимуму всего горизонта. Если хотите зону-по-зоне — используется `referenceArea` (см. Решения агента §12.1).

**Tooltip (CustomForecastTooltip):**

```
┌────────────────────────────────┐
│  15 января 2026                │
│  Net Worth: 185 000 ₽          │
│  ─────────────────────         │
│  Ближайшие операции:           │
│  💰 ЗП               +80 000 ₽│
│  🏠 Ипотека          −45 000 ₽│
│  🛒 Продукты          −8 000 ₽│

│  USD: $450 | EUR: €200         │
```

**Линия сценария:**

**Loading state:** Skeleton высотой 300px + Skeleton для контролов.

- Отступ слева `pl="lg"` (индикация вложенности)
- Таблица `byCategory` из ответа API: категория → сумма
- Группировка: Доходы отдельно, Расходы отдельно
- `fz="sm"`, `c="dimmed"` для названий категорий
- Если категорий нет — скрыть строку

**Loading state:** `Skeleton` на месте таблицы, 6 строк.

### 5.5 Alerts Panel (Правая колонка)

**Компонент:** `Paper` (`p="md"`, `withBorder`, `radius="md"`) + `Stack`

**Заголовок:**
```
Предупреждения   [3]
```
- "Предупреждения": `fz="sm"`, `fw={600}`, `c="dimmed"`, `tt="uppercase"`
- `[3]` — `Badge` с количеством алертов `severity="critical"`, `color="red"`, `variant="filled"`
- Если нет critical — Badge не отображается

**Каждый алерт (Alert из Mantine):**

| Тип severity | color | icon | variant |
|---|---|---|---|
| critical | red | 🚨 (IconAlertOctagon) | light |
| warning | yellow | ⚠️ (IconAlertTriangle) | light |
| info | blue | ℹ️ (IconInfoCircle) | light |

Структура каждого Alert-компонента:
```
┌──────────────────────────────────────────┐
│  🚨  Критично                            │
│      Баланс счёта "Дебетовая" упадёт ниже│
│      нуля 15 июля 2026                   │
│                                          │
│  [развернуть ▼]                          │
│  ─────────────────                       │
│  Рекомендация: Пополните счёт или        │
│  перенесите плановые расходы             │
└──────────────────────────────────────────┘
```

**Поведение алерта:**
- По умолчанию: показывает только заголовок (тип + короткий Message)

**Hash-функция для цвета тега:**
```js
const TAG_COLORS = ['blue', 'cyan', 'teal', 'green', 'lime', 'yellow', 'orange', 'red', 'pink', 'grape', 'violet', 'indigo'];
function tagColor(name) {
  let hash = 0;
  for (const char of name) hash = (hash * 31 + char.charCodeAt(0)) % TAG_COLORS.length;
  return TAG_COLORS[hash];
}
```

## 12. Решения агента

### Решение 1: Цвет заливки Forecast Chart — по минимуму, не по зонам

**Инструкция:** заливка области меняет цвет в зависимости от значения (зелёная > 50000, жёлтая 0-50000, красная < 0).

**Принято:** Recharts нативно не поддерживает изменение цвета заливки Area по значению Y. Реализовать это через несколько `<Area>` с разными `<clipPath>` или `<ReferencArea>` возможно, но крайне сложно и нестабильно.

**Выбрано:** единый цвет заливки определяется по **минимальному значению** в отображаемом горизонте:
- Если `min < 0` → красный
- Если `min >= 0 && min < 50000` → жёлтый
- Если `min >= 50000` → зелёный

`ReferenceLine` для y=0 и y=50000 остаются — они дают визуальное разграничение зон. Это менее красиво, но стабильно.

### Решение 2: Тип операции (Доход/Расход) как SegmentedControl, не знак Amount

**Инструкция:** Amount в БД хранит знак (положительный = доход, отрицательный = расход).

**Принято:** в форме ввода пользователь всегда вводит положительное число, а тип (Доход/Расход) выбирает через `SegmentedControl`. Знак автоматически проставляется при сохранении. Это привычнее для пользователя (не вводить -80000).

### Решение 3: Inline-редактирование категорий

**Инструкция:** упомянута CRUD-таблица, но способ редактирования не уточнён.

**Принято:** указаны оба варианта:
1. Кнопка ✏️ → inline-редактирование прямо в строке таблицы (предпочтительно, меньше переходов)
2. Альтернативно — Modal (если inline неудобен на мобильных)

Финальное решение за разработчиком, но рекомендован inline-режим как более удобный.

### Решение 4: Навигация по карточке аккаунта — клик на всю карточку

**Инструкция:** упомянуто "клик по карточке → детальная страница", но не уточнено взаимодействие с кнопками внутри.

**Принято:** вся карточка кликабельна (cursor: pointer, Navigate on click). Кнопки footer ("Редактировать", "Архивировать") вызывают `e.stopPropagation()` чтобы не активировать переход.

### Решение 5: Относительные даты в Upcoming Operations ограничены 7 днями

**Инструкция:** форматирование дат — "через 3 дня", "вчера".

**Принято:** относительные даты показываются только для дат в диапазоне [-3, +7] дней от сегодня. За пределами — полный формат "15 янв". Это не перегружает UI непривычными форматами для далёких дат.

### Решение 6: Теги хранятся нормализованными (lowercase, trim)

**Инструкция:** описание `Tags` в PlannedOperations — "массив строк".

**Принято:** при сохранении операции теги нормализуются: `tag.trim().toLowerCase()`. В UI отображаются в lowercase. Цвет вычисляется по нормализованному имени. Это обеспечивает консистентность: "Ипотека" и "ипотека" — один тег.

### Решение 7: Горизонт прогноза — глобальный state (Zustand)

**Инструкция:** переключатель горизонта 3/6/12 мес на Dashboard.

**Принято:** выбранный горизонт хранится в глобальном Zustand store (не в локальном state компонента). Это позволяет использовать тот же горизонт на экране Scenarios (для расчёта дельты net worth) и других местах без передачи через props.

### Решение 8: Кнопка "Сохранить" в Settings → Профиль сохраняет все поля разом

**Инструкция:** не уточнено, сохраняется ли каждое поле отдельно или все вместе.

**Принято:** одна кнопка "Сохранить" в конце формы, отправляет PATCH с `{ DisplayName, BaseCurrency }`. Email — readonly, не отправляется. Тема сохраняется в localStorage немедленно без кнопки (т.к. это локальная настройка браузера).

### Решение 9: Drag-and-drop счетов вынесен за скоуп MVP экрана /accounts

**Инструкция:** упомянут `SortOrder` в Accounts.

**Принято:** DnD-сортировка счетов не реализована в MVP экране /accounts (только в API схеме присутствует SortOrder). Счета отображаются в порядке `SortOrder` (который выставляется вручную через API), но UI для перетаскивания карточек не делается — слишком высокая сложность для MVP карточек. DnD реализован только для категорий (в Settings → Категории), где он проще в таблице.

### Решение 10: Компактный вид "Upcoming Operations" — только первый тег

**Инструкция:** "теги (Badge, xs)" — не уточнено, сколько тегов показывать.

**Принято:** в виджете "Ближайшие операции" показывается только первый тег. Остальные скрыты (нет "+N ещё" — слишком мало места). Полный список тегов виден при открытии операции в модале.
