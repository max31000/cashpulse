# Алгоритм начисления процентов по вкладам и сберегательным инвестсчётам

Версия: 1.0  
Дата: 2026-04-05  
Относится к: `ForecastEngine` — метод `GenerateInterestAccruals()`

---

## Область применения

Алгоритм применяется к счетам следующих типов:
- `AccountType.Deposit` — любой вклад
- `AccountType.Investment` с `InvestmentSubtype == "savings"` — сберегательный инвестсчёт

Для `Investment` с подтипами `bonds` и `stocks` — не применяется (доходность не детерминирована).

---

## Входные данные

```csharp
Account account          // Счёт с полями: InterestRate, InterestAccrualDay,
                         // DepositEndDate, CreatedAt
decimal initialBalance   // Баланс счёта на начало расчёта (из CurrencyBalances)
string currency          // Основная валюта счёта
DateOnly from            // Начало горизонта (HorizonStart)
DateOnly to              // Конец горизонта (HorizonEnd)
```

Предусловия:
- `account.InterestRate != null && account.InterestRate > 0`
- `account.InterestAccrualDay != null`
- `initialBalance > 0` (проценты начисляются только на положительный баланс)

---

## Выходные данные

```csharp
List<ProjectedOperation>
// Каждый элемент — одно начисление процентов с IsAutoGenerated = true
```

Структура `ProjectedOperation` расширяется флагом автогенерации:

```csharp
public record ProjectedOperation
{
    public DateOnly Date { get; init; }
    public decimal Amount { get; init; }
    public string Currency { get; init; }
    public long AccountId { get; init; }
    public long? CategoryId { get; init; }
    public List<string>? Tags { get; init; }
    public string? Description { get; init; }
    public long TemplateOperationId { get; init; }
    public long? ScenarioId { get; init; }
    public bool IsRecurring { get; init; }
    public bool IsAutoGenerated { get; init; }  // NEW: true для авто-операций
}
```

## Алгоритм (псевдокод)

```
ФУНКЦИЯ GenerateInterestAccruals(account, initialBalance, currency, from, to):

    IF account.InterestRate == NULL OR account.InterestRate <= 0:
        RETURN []
    
    IF account.InterestAccrualDay == NULL:
        RETURN []
    
    IF initialBalance <= 0:
        RETURN []
    
    accrualDay = account.InterestAccrualDay  // гарантировано 1-28 на уровне БД
    effectiveTo = min(to, account.DepositEndDate ?? to)
    
    results = []
    
    // Перебираем месяцы горизонта
    currentMonth = (from.Year, from.Month)
    lastMonth    = (effectiveTo.Year, effectiveTo.Month)
    
    ПОКА currentMonth <= lastMonth:
        year  = currentMonth.Year
        month = currentMonth.Month
        
        // --- Шаг 1: Определить дату начисления ---
        daysInMonth = DateTime.DaysInMonth(year, month)
        
        // Fallback: если день > кол-ва дней в месяце (хотя ограничено 1-28, для safety)
        accrualDate = new DateOnly(year, month, min(accrualDay, daysInMonth))
        
        // Пропустить если дата начисления раньше начала горизонта
        IF accrualDate < from:
            currentMonth = NextMonth(currentMonth)
            CONTINUE
        
        // Пропустить если дата начисления позже effectiveTo
        IF accrualDate > effectiveTo:
            BREAK
        
        // --- Шаг 2: Определить период начисления ---
        // Предыдущая дата начисления (или дата начала горизонта для первого периода)
        prevYear  = year
        prevMonth = month - 1
        IF prevMonth < 1:
            prevYear  = year - 1
            prevMonth = 12
        
        prevDaysInMonth = DateTime.DaysInMonth(prevYear, prevMonth)
        prevAccrualDate = new DateOnly(prevYear, prevMonth, min(accrualDay, prevDaysInMonth))
        
        // periodStart = предыдущая дата начисления (или from для первого раза)
        periodStart = max(prevAccrualDate, from)
        periodEnd   = accrualDate
        days        = periodEnd.DayNumber - periodStart.DayNumber
        
        IF days <= 0:
            currentMonth = NextMonth(currentMonth)
            CONTINUE
        
        // --- Шаг 3: Сумма начисления ---
        // MVP: используем начальный баланс (не учитываем изменения внутри периода)
        dailyRate = account.InterestRate / 100.0m / 365.0m
        accrual   = initialBalance * dailyRate * days
        accrual   = Math.Round(accrual, 2, MidpointRounding.AwayFromZero)
        
        IF accrual <= 0:
            currentMonth = NextMonth(currentMonth)
            CONTINUE
        
        // --- Шаг 4: Создать ProjectedOperation ---
        results.Add(new ProjectedOperation
        {
            Date             = accrualDate,
            Amount           = accrual,
            Currency         = currency,
            AccountId        = (long)account.Id,
            CategoryId       = null,          // авто-операции без категории
            Tags             = null,
            Description      = "Начисление процентов (авто)",
            TemplateOperationId = 0,          // нет шаблона в PlannedOperations
            ScenarioId       = null,
            IsRecurring      = true,
            IsAutoGenerated  = true
        })
        
        currentMonth = NextMonth(currentMonth)
    
    RETURN results
```

## C# реализация в ForecastEngine

```csharp
/// <summary>
/// Генерирует авто-операции начисления процентов для вклада или сберегательного инвестсчёта.
/// Авто-операции не сохраняются в PlannedOperations — генерируются на лету в ForecastEngine.
/// </summary>
private List<ProjectedOperation> GenerateInterestAccruals(
    Account account,
    decimal initialBalance,
    string currency,
    DateOnly from,
    DateOnly to)
{
    if (account.InterestRate is null or <= 0) return [];
    if (account.InterestAccrualDay is null) return [];
    if (initialBalance <= 0) return [];

    var accrualDay  = account.InterestAccrualDay.Value;
    var effectiveTo = account.DepositEndDate.HasValue
        ? DateOnly.FromDayNumber(Math.Min(to.DayNumber, account.DepositEndDate.Value.DayNumber))
        : to;

    var results = new List<ProjectedOperation>();

    var year  = from.Year;
    var month = from.Month;

    while (new DateOnly(year, month, 1) <= new DateOnly(effectiveTo.Year, effectiveTo.Month, 1))
    {
        // Шаг 1: дата начисления с fallback на последний день месяца
        var daysInMonth = DateTime.DaysInMonth(year, month);
        var actualDay   = Math.Min(accrualDay, daysInMonth);
        var accrualDate = new DateOnly(year, month, actualDay);

        if (accrualDate < from)
        {
            AdvanceMonth(ref year, ref month);
            continue;
        }

        if (accrualDate > effectiveTo)
            break;

        // Шаг 2: предыдущая дата начисления для определения периода
        var (prevYear, prevMonth) = month == 1
            ? (year - 1, 12)
            : (year, month - 1);

        var prevDaysInMonth = DateTime.DaysInMonth(prevYear, prevMonth);
        var prevAccrualDate = new DateOnly(prevYear, prevMonth, Math.Min(accrualDay, prevDaysInMonth));

        var periodStart = prevAccrualDate < from ? from : prevAccrualDate;
        var days        = accrualDate.DayNumber - periodStart.DayNumber;

        if (days <= 0)
        {
            AdvanceMonth(ref year, ref month);
            continue;
        }

        // Шаг 3: расчёт суммы
        // Упрощение MVP: используем initialBalance (баланс на начало горизонта)
        var dailyRate = account.InterestRate.Value / 100m / 365m;
        var accrual   = Math.Round(initialBalance * dailyRate * days, 2, MidpointRounding.AwayFromZero);

        if (accrual <= 0)
        {
            AdvanceMonth(ref year, ref month);
            continue;
        }

        // Шаг 4: создать ProjectedOperation
        results.Add(new ProjectedOperation
        {
            Date                = accrualDate,
            Amount              = accrual,
            Currency            = currency,
            AccountId           = (long)account.Id,
            CategoryId          = null,
            Tags                = null,
            Description         = "Начисление процентов (авто)",
            TemplateOperationId = 0,
            ScenarioId          = null,
            IsRecurring         = true,
            IsAutoGenerated     = true
        });

        AdvanceMonth(ref year, ref month);
    }

    return results;
}

private static void AdvanceMonth(ref int year, ref int month)
{
    if (++month > 12)
    {
        month = 1;
        year++;
    }
}
```

## Место вызова в ForecastEngine.Calculate()

Авто-операции добавляются в `allProjectedOperations` **после** `ExpandAllOperations()`, **до** построения таймлайнов:

```csharp
// Step 1: Expand all recurrences + collect one-time ops
var allProjectedOperations = ExpandAllOperations(request);

// Step 1b: Inject interest accruals for deposits and savings
foreach (var (accountId, account) in request.Accounts)
{
    bool isDepositOrSavings =
        account.Type == AccountType.Deposit ||
        (account.Type == AccountType.Investment && account.InvestmentSubtype == "savings");

    if (!isDepositOrSavings) continue;

    // Берём баланс по основной валюте счёта (первая валюта в Balances)
    var mainBalance = account.Balances.FirstOrDefault();
    if (mainBalance is null) continue;

    var accruals = GenerateInterestAccruals(
        account,
        initialBalance: mainBalance.Amount,
        currency:       mainBalance.Currency,
        from:           request.HorizonStart,
        to:             request.HorizonEnd
    );

    allProjectedOperations.AddRange(accruals);
}

// Step 2: Build account timelines
var accountTimelines = BuildAccountTimelines(request, allProjectedOperations);
// ... остальные шаги без изменений
```

## Ограничения и допущения MVP

| Ограничение | Обоснование |
|-------------|-------------|
| Баланс не пересчитывается внутри периода | Упрощение MVP. Для точного расчёта нужно учитывать пополнения/снятия, что усложняет алгоритм. |
| 365 дней в году (не 365/366) | Упрощение. Банки используют разные конвенции (ACT/365, ACT/ACT). Для MVP разница несущественна. |
| `TemplateOperationId = 0` | Авто-операции не имеют соответствующей записи в PlannedOperations. `0` — условный маркер (не `null` для совместимости с типом `long`). |
| Авто-операции не попадают в Monthly Breakdown расходов | Начисление процентов — доход (+), участвует в Net Worth. Включается в `Income` monthly breakdown. |

## Пример расчёта

**Входные данные:**
- `InterestRate = 16.00` (16% годовых)
- `InterestAccrualDay = 15`
- `initialBalance = 500 000.00 RUB`
- `from = 2026-04-05`, `to = 2026-06-30`
- `DepositEndDate = null`

**Итерации:**

| Месяц | accrualDate | periodStart | periodEnd | days | accrual (руб.) |
|-------|-------------|-------------|-----------|------|----------------|
| Апрель | 2026-04-15 | 2026-04-05 (=from) | 2026-04-15 | 10 | 500 000 × 0.16 / 365 × 10 = **219.18** |
| Май | 2026-05-15 | 2026-04-15 | 2026-05-15 | 30 | 500 000 × 0.16 / 365 × 30 = **657.53** |
| Июнь | 2026-06-15 | 2026-05-15 | 2026-06-15 | 31 | 500 000 × 0.16 / 365 × 31 = **679.45** |

Итого за период: **1 556.16 RUB**
